> IMPORTANT
>
> Pull Requests without an issue linked may not be reviewed. Open or get an issue assigned first, then submit your PR.

## Prerequisites

- Xcode 26+
- Swift 5+
- iOS Simulator or physical iPhone for validation
- Git

## How to Contribute

1. Review existing issues or create a new one.
2. Fork the repository and create a branch for your change.
3. Implement your fix/feature with focused commits.
4. Test in Xcode (build + run).
5. Open a Pull Request and link the issue.

## Pull Request Guidelines

- Keep PRs scoped to one logical change.
- Include screenshots for UI changes.
- Update documentation where required.
- Ensure CI passes.
- Be responsive to review feedback.

If you need help, open a discussion in issues.
